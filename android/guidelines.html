User Interface Code Specifications

Short Term Goal

To write out a user interface for a SIPDroid or XMPP stack on the Android 2.0 platform.

Long Term Goal

Develop a modular architecture for an user interface for a chat system that could be cross-platform that will be developed by multiple users

Implications of Long Term

  • Cross-Platform means to avoid all environment specific settings or code. Minimize system and OS dependencies, or isolate them in their own modules to prevent the need for re-development
  • Cross-Platform also means cross-GUI toolkit. Make sure that the views for objects are also isolated from the controls and models, so that the algorithms may remain, with swappable views for each environment.
  • Standards based coding so that multiple users can understand the code. Java syntax is preferred.
  • Documentation must be kept up to date. No exceptions after a push.
  • We will be using the MVC (Model-View-Controller) method for the reasons listed
  • Support for generics is in Java 1.5+, and should be used to prevent casting errors in the future
  • Hacks are useful to get things working. But comment them so that people are aware that its a hack.

Coding Standards

  • Java convention means that classes are capitalized
  • Convention has variables starting with lower-case
  • Javadoc is tedious but standard
  • Constants may be capitalized or all-caps, but remain consistent in classes
  • Packages should start with org.opencomm
